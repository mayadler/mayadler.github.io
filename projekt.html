About the Project

What is 2 to 3 Weeks?

In professional soccer, "2 to 3 weeks" is the most common phrase uttered when a player gets injured — a vague prognosis that leaves coaches, clubs, and fans in the dark. We decided to take that phrase and turn it into a prediction engine.

Our web application ingests player biometric data, workload metrics, historical injury records, and match statistics to output a real-time injury risk score. The goal is to give performance staff the early warning they need to rotate squads, adjust training loads, and prevent career-altering injuries before they happen.

Built over four sprint cycles with a full-stack architecture, the system pairs a React frontend with a Python ML backend (scikit-learn + FastAPI), trained on anonymised player datasets.

  • ⚡
    Real-time Risk Scoring
    Predict injury probability per player based on live match and training data streams.
  • 🧠
    ML-Powered Prediction
    Gradient-boosted model trained on 5,000+ player-seasons of professional match data.
  • 📊
    Coach Dashboard
    Intuitive visual interface showing squad risk levels, trends, and actionable recommendations.
  • 🔒
    Privacy-First Design
    All player data is anonymised and stored securely in compliance with GDPR standards.

Development

Sprint Cycles

The project was built using Agile methodology across four two-week sprint cycles, each delivering a shippable increment of the product.

Sprint 01 · Weeks 1–2
Foundation & Research
Established the technical stack, conducted domain research, gathered and cleaned the initial dataset, and set up the project repository and CI/CD pipeline.
  • Dataset collection & preprocessing
  • Literature review on injury biomechanics
  • Tech stack decision (React + FastAPI)
  • Repository setup & branching strategy
Completed
Sprint 02 · Weeks 3–4
ML Model Development
Built and validated the core machine learning pipeline. Explored multiple algorithms including logistic regression, random forests, and gradient boosting before settling on XGBoost.
  • Feature engineering & selection
  • Baseline model benchmarking
  • XGBoost tuning & cross-validation
  • Model explainability (SHAP values)
Completed
Sprint 03 · Weeks 5–6
Frontend & API Integration
Designed and implemented the coach-facing dashboard. Connected the React frontend to the FastAPI backend and built the data ingestion pipeline for real-time player metrics.
  • Dashboard UI/UX design & build
  • REST API endpoints for predictions
  • Player profile management module
  • Data visualisation components
Completed
Sprint 04 · Weeks 7–8
Testing, Polish & Expo Prep
End-to-end testing, accessibility improvements, performance optimisation, and preparation of demo materials and documentation for the final expo presentation.
  • Unit & integration test suite
  • Performance & load testing
  • Demo data preparation
  • Poster & presentation design
